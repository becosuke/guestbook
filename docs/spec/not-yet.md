# Not Yet

現状の Guestbook で **まだ提供していない** こと。
未着手の項目を「検討中」と「将来的な余白」に分けて整理する。

- **検討中** — プロダクトとして次に取り組みたい方向として既に視野に入っているもの
- **将来的な余白** — 必要になったときに考える、現時点では優先度が決まっていないもの

---

## 検討中

### テーマ別の「板」を分けたい

- 現状は投稿が **1 つのタイムライン** にすべてまとまっている。
- これを「雑談」「お知らせ」「質問」のような **テーマごとの板** に分けられるようにしたい。
- 検討ポイント:
  - 板は誰が作れるのか（運営だけ／ユーザーが自由に作れる）
  - 板の数が増えたときの一覧・発見の体験
  - 投稿を別の板に移動できるかどうか

### 投稿に「いいね」を付けたい

- 読み手が「この投稿を支持する」「面白かった」といった気持ちを軽く表明できる仕組み。
- 検討ポイント:
  - **「Bad（低評価）」も付けられるようにするかは未定**。意見表明の幅が広がる一方で、攻撃的に使われやすい性質もあるため要検討。
  - いいねの数を投稿の並び順に反映するか（新着順固定のまま並べるか）

### マイページ

- 自分の投稿、自分が「いいね」した投稿、自分がフォローしている対象などを 1 箇所で見渡せるページ。
- 投稿者の識別ができるようになることが前提。
- 検討ポイント:
  - 公開する／しないの切り替え（プロフィールページとして公開するか、自分用のダッシュボードに留めるか）

### ユーザーや板のフォロー

- 気に入ったユーザーや板を **フォロー** して、新しい投稿を追いかけやすくしたい。
- マイページに「フォロー中の人・板の最新投稿」がまとまる体験を想定。
- 検討ポイント:
  - フォロー関係を公開するか／非公開にするか
  - フォロー対象から外す（ミュート／ブロック）操作の扱い

## 将来的な余白

### 投稿者の識別がない（匿名のみ）

- 現在は誰でも匿名で書き込めるかわりに、「誰の投稿か」をサービスが把握していない。
- 結果として、**他人の投稿を書き直したり取り下げたりすることも技術的には可能** になっている。
  「自分の投稿だけ自分が操作できる」状態は現状保証されない。
- マイページ・フォロー・いいねといった「検討中」の機能は、いずれもこの基盤が必要になる。
- 想定される対応の方向:
  - ログイン機構（メール／OAuth など）を導入し、投稿に投稿者を紐付ける
  - 投稿時に「あなたしか知らない編集キー」を発行し、書き直し・取り下げ時に提示させる軽量な方式

### 特定の投稿への返信（レス）がない

- 一般的な掲示板にある「○○番に返信」のような **スレッド／レス機能は無い**。
- 投稿同士はすべて並列で、親子関係を持たない。
- 想定される対応:
  - 投稿に「どの投稿への返信か」を持たせ、一覧をスレッド単位で見られるようにする
  - 返信の連鎖をどこまで辿るか（1 段だけ／無制限）を決める

### 不適切投稿への対処手段がない

- 通報、ブロック、モデレーション、NG ワードフィルタといった機能は提供していない。
- 全ての投稿は書かれた内容のまま、誰でも読める状態になる。

### 検索・絞り込みがない

- 投稿は新着順で一覧表示されるのみ。
- キーワード検索、日付での絞り込み、ID 以外による検索は提供していない。

### 通知がない

- 自分の投稿に反応があった、フォロー中のユーザーが投稿した、といった **通知の仕組みは無い**。
- いいね・フォロー機能を入れる際には、それらと結びついた通知の体験も合わせて設計する想定。

### 公開範囲の調整がない

- 投稿は常に **全員に公開** される。
- 「自分だけ」「特定の人だけ」のような公開範囲を絞る設定は無い。
